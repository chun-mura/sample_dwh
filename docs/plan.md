実際に手を動かしてデータ分析基盤の構築を学ぶために、本書の第9章で紹介されている**「売上データの可視化プロジェクト」**を題材にした学習プランを提案します。

Webエンジニアとしてのスキルを活かしつつ、データの収集から可視化までを一気通貫で体験できる内容です。

### 1. サンプルデータの定義
まずは、分析の対象となる2つのデータソースを準備（あるいはシミュレーション）します。

*   **売上データ（Sales Data）**: 毎時APIから取得される想定のトランザクションデータです。
    *   `purchase_id`: 購入を一意に識別するID（品質テストで重複チェックに使用）。
    *   `store_name`: 店舗名（「Store A」「Store B」など）。
    *   `sales`: 売上金額。
    *   `scale`: 通貨単位（「yen」または「us_dollar」）。
    *   `time`: 売上発生日時。
*   **為替データ（Currency Rates）**: 1日1回取得する、円建て計算のためのマスタデータです。
    *   `currency`: 通貨（JPYなど）。
    *   `rates`: 1ドルあたりのレート。

### 2. データ基盤の4レイヤー構成
このデータを以下の4つの階層（レイヤー）に沿って流していきます。

1.  **コレクティング（収集）**: Pythonスクリプト等を用い、APIからJSON形式でデータを取得し、Amazon S3の**「ローゾーン」**にそのまま保存します。
2.  **プロセシング（処理）**: AWS Glue（Apache Spark）を使用します。ここで「ドルから円への換算」「`purchase_id`のユニーク性チェック（データ品質テスト）」を行い、分析しやすい**Parquet形式（Snappy圧縮）**に変換します。
3.  **ストレージ（保存）**: 加工済みのデータをS3の**「ゴールドゾーン」**に保存します。この際、日付や店舗ごとに**パーティション**を分けることで、クエリの高速化を図ります。
4.  **アクセス（提供）**: **Amazon Athena**を使ってSQLでデータを参照し、最終的に**Amazon QuickSight**などのBIツールで「店舗別・時間別の売上ダッシュボード」を作成します。
